Stego

What is this?

Steganography is basically a way to conceal message, file or picture inside a file. The reason why I am writing about this is because many of the challenges you find in CTF are “stego”. For example, you receive a picture, where you either have to look what’s inside the file or go through filters to uncover the hidden text.

How do I decode or encode?

By doing many CTF’s and failing over and over again, you will come across many tools. Some of them are useful, and some of them are not. I will try to document every tool that I come across, so you can try to connect which tool works out for the file you have.

  • Make sure you always know which file type you are working with by using file in terminal.

file README.txt

README.txt: ASCII text

  • Something that you have to get used to is to use the strings command, to check if there is something valuable text in the file.
MP3 files

There are a couple of different ways to hide messages inside MP3 file, I will be focusing more on spectrogram. Many use audacity to analyze the file

open audacity –> open the file –> drop down menu –> select spectrogram.

Personally I like to always check the easiest way, then analyze it more specific if the easiest way did not work. A very cool website that analyzes the MP3 file is academo, it is easy and fast.

example of a what it looks like when you analyze the file.

PNG files

If you checked for strings, and did not find anything valuable the next thing will be to check the png for different filters.

Stegsolve.jar is a very good tool to go through different filters and colors, you only have to upload the picture to the java application, and go through the colors and filters with your arrow keys.
