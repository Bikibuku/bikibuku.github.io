Elementary searching methods


Get the keys and sort them asap

In these type of exercises you often get some keys, in the order left to right where space does not count. It is not hard, but I recommend to draw the tree and always keep some extra space in case you mess up. I will go through heap, binary search tree, 2-3-4 tree and red black tree.

The keys are: M A N C H E S T E R

Heap

All you have to do here is start with the keys (left to right) and fill in the tree. Is the key higher value than the “dad”? If yes, change the position.

The answer here, in level order: TSNHREMAEC

binary search tree

The only rules you need:

  • The root is the first key of the keys (M in this case).
  • If the key is greater or equal, go right.
  • If the key is less, go left.
