Subnetting

Important things to know before learning

Subnetting is a bit complicated, here we will learn how to find the subnet address, 1st host address, last host adress and the broadcast address by knowing the IP address. There are some methods which is quicker, but I feel once you mastered some binary tasks, you will be able to do this quicker. I feel this is a lot more safer and more efficient way to do it.

Binary rules

Network / subnet address:

  • Fill the host portion of an address with binary 0’s

Broadcast address:

  • Fill the host portion of an address with binary 1’s

First host address:

  • Fill the host portion of an address with binary 0’s except for the last bit which is set to binary 1

Last host address:

  • Fill the host portion of an address with binary 1’s except for the last bit which is set to binary 0

Basic example

IP address: 192.168.1.18 /24
Mask: 255.255.255.0
24 means 24 bits in the subnet mask, we have 8 bits in every octet.

So we will be working on the last octet, remember the binary rules. In the single octet you will be entering the binary, the answer will be in decimal.

Single octet Answer
Subnet 192.168.1.00000000 192.168.1.0
1st host 192.168.1.00000001 192.168.1.1
last host 192.168.1.11111110 192.168.1.254
Broadcast 192.168.1.11111111 192.168.1.255

What if the mask is not /8,/16,/24?

Let’s say we got IP address of 172.16.35.123 /20 (255.255.240.0). What you need to do, is to convert it to binary. The first two octet results in 16, so you kind of know that you need to work with the third octet.

We have converted the decimal, 35, to binary. The green line indicates the host portion and network portion.

Single octet Answer
Subnet 172.16.00100000.00000000 172.16.32.0
1st host 172.16.00100000.00000001 172.16.32.1
last host 172.16.00101111.11111110 172.16.47.254
Broadcast 172.16.00101111.11111111 172.16.47.255
