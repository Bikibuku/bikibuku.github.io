OSI model

OSI model - open system interconnection model is a critical building block for your understanding of networking. It helps you troubleshoot networks, if the cable isn’t plugged (Physical - layer 1).

Layers

It consist of layers shown in the figure beneath this text, and the layers are read bottom up. Application, presentation and session layer is the primary focus of developers / SysAdmin / ServerAdmin.

Easy way to memorize the layers is to remember the sentence “All people sleeping through networking don’t pass”


Advantages

There are multiple advantages to the OSI model:

  • Standard(ISO) and interoperability (the ability of computer systems or software to exchange and make use of information).
  • Split development (developer working in layer 7, does not need to know what is happening in layer 2).


Layer 7 - Application layer

This layer is very familiar for you, where applications such as HTTP, FTP and TELNET are used at this layer. You have to note that the application layer is not about individual applications (Chrome/Firefox), it is about the application protocol (HTTP that web browsers use). The layer is the for users and application process to access network recourses and services.

So what do the application layer do?

  • This layer provides network processes to application.
  • It provides access for users and application processes to utilize network services.
  • This layer provides the network services to the application (web browsers)
  • It can identify communication partners
  • Provide user authentication

So what does that mean? The simplest way to explain this is: Picture Microsoft Word, it works with many different protocols, and then use the protocols to communicate with the lower layers of the OSI model.

Other examples:

  • Remote file access
  • Remote printer access
  • Resource sharing
  • Directory services

Layer 6 - Presentation layer

This layer is concerned with the actual format the data is presented in i.e., we are looking at data representation / syntax. Presentation layer has to ensure that data is readable by receiving system. Data that is sent from one application can be successfully read by another application.

You might think that different system format their data different, and you are right. The data needs to be changed to machine independent format (ASCII, MP3, JPG). So this layer formats data to be presented to application layer.

The layer also structures the data, negotiates data transfer syntax (using ASCII or unicode). It can also provide encryption: if you are sending music file from a device to another one, this layer will assure that the data is transmitted in the format that both sides understand.

Layer 5 - Session layer

The session layer is concerned with interhost communication. This layer is involved in the maintenance, establishment and termination of sessions between applications. This allows two application processes on different machines, to establish a session between them. This acts like a coordinator of communication between systems. So various protocols at this layer ensure that request and responses between applications can work properly over a session, but between those end-user application processes.

Indicates session layer:

  • Security
  • Name recognition
  • Logging

Layer 4 - Transport layer

This layer ensures end to end communication, reliability and flow control. It also performs message segmentation, it accepts a message from the session layer, splits the message into smaller units and pass it down to the network layer for sending. Session multiplexing takes place at this layer, this allows for multiplexing several message streams or sessions into one logical link, this layer will keep track of which message belongs to which session. Flow control might be implemented in this layer, it will manage data transmission between devices to ensure that the transmitting device does not send more data than the receiving device can process.

Other things that transport layer does:

  • Ensures data transport reliability
  • Handles transportation issues between host

Two main protocols at this layer is TCP (Transmission control protocol) and UDP (User Datagram Protocol), please check the TCP/UDP post for more information.

TCP:

  • Provides reliability
  • Establishes, maintains and terminates virtual circuits between devices.
  • So end systems will go through what’s called the TCP handshake to establish a connection between each other
  • If the packets is missing it will be retransmitted
  • Acknowledgement of receipt of information

UDP:

  • Does not provide reliability
  • If packets are dropped, they are lost
  • Does not retransmit lost packets
  • “Lightweight”

Layer 3 - Network layer

This layer is all about data delivery, where routers route data packets from one device to another. Cisco have something called layer 3 switches, they have router capabilities so that they can also route packet between VLANS within a network as an example. They chose the best path to deliver data based on the information provided. Here we have routing protocols like OSPF (Open shortest path first), BGP (border gateway protocol) or IS-IS (intermediate system-to-intermediate system). They use the logical addressing scheme of IPv4 to determine the best path.

Criteria to determine the best path:

  • Cost
  • Hop count
  • Bandwith
  • Longest match of network address

In conclusion, this layer provides path determination and logical addressing.

Layer 2 - Data link layer

Here it provides physical addressing and access to media, it is concerned how data is formatted from upper layers for transmission over a given network technology. It is also concerned with how access to the network is controlled.
The data link layer ensures that the data from higher layers is formatted correctly for transmission across individual physical links, where the physical links characteristics may vary (one link may be ethernet, PPP across a serial cable and other link may use DSL)

Layer 1 - Physical layer

The first layer in the OSI model is the physical layer, it defines here how data is transmitted So in other words, what state represents a 0 or a 1. It defines the electrical, mechanical, procedural and functional specifications for activating, maintaining and deactivating the physical link. This allows for interoperability. So long story short, it is about how to code/decode series of bits from and to the physical device (cable etc..).
